# VoiceFlow：纯语音 AI 绘图工具

VoiceFlow 面向比赛题目“AI 语音绘图工具”，用户授权浏览器麦克风后，可以不使用鼠标、键盘或触摸编辑，通过语音完成专业图表与自由图形的创建、修改、排版、版本管理和导出。

> [观看演示视频：VoiceFlow——一个 AI 语音绘图系统](https://www.bilibili.com/video/BV1f1JK6eECB/?share_source=copy_web&vd_source=b8bfc1e86705610f3e025227e4fbac96)
>
> [查看 VoiceFlow 项目说明书 PDF](VoiceFlow项目说明书.pdf) · [阅读技术方案设计](VoiceFlow%20技术方案设计.md)

语音层持续消费 ASR 的 interim/final 结果，在用户连续说话时识别语义断句并生成严格有序的任务队列。边界明确、低风险的队首任务可以边听边执行；复杂任务等待本轮录音结束，任何后续简单任务都不能越过前面的任务。

界面采用左侧语音工作区与右侧画布区。工作区展示麦克风与实时字幕、严格有序的任务队列、文字指令测试和版本管理；画布可以在“专业图表”和“自由画图”两种模式之间切换。

![VoiceFlow 参赛界面](docs/images/voiceflow-overview.png)

页面默认不监听麦克风。点击“开始语音输入”后开始持续识别，完成后点击“停止语音输入”立即关闭监听。

无法使用麦克风时，可在左侧“文字指令测试”输入框中输入命令并按 Enter 或点击“执行指令”。文字测试会复用与语音最终识别结果相同的路由和执行链路。

## 核心创新

### 分层低延迟路由

```text
语音输入
  -> Fast Path：撤销、保存、导出、缩放等常用命令本地直通
  -> Simple Path：确定性的节点与连线操作
  -> Workflow Path：版本、主题、场景和视图工作流
  -> Agent Path：完整结构图规划与无法确定的上下文修改
  -> Local Planner：未配置 AI 时的完整结构图降级
```

系统不会让所有命令都等待 AI。确定性命令直接本地执行，界面实时展示执行路径、路由置信度、本次耗时、各路径平均耗时和 Fast Path 命中率。

### 本地低延迟 ASR 校准

语音校准不调用 AI，也不发起网络请求。系统使用可扩展的多来源词典，综合内置绘图词汇、命令别名、常见错词、拼音混淆、编辑距离、当前画布节点/连线和最近命令生成候选。候选经过加权排序、最佳/次佳差值门槛和重叠冲突消解后才会替换，避免词典扩大后误改未知业务词。

当前内置基线包含 106 个规范词、51 个别名、19 条高精度错词规则和 71 组常见同音混淆，并使用 `pinyin-pro` 为任意中文业务词生成完整拼音。运行时词典会自动加入当前图标题、所有节点、连线标签和最近命令，因此不受内置词表数量上限约束。

### LLM 主导的通用结构图规划

完整新图由已配置的 LLM 根据用户主题生成紧凑节点关系蓝图，Prompt 会约束图表类型、节点类型、关系质量和输出 JSON 协议，避免模型输出坐标与任意样式。本地校验器负责补全 ID、拒绝非法关系，并交给 `ELK` 与 `React Flow` 完成布局和渲染。未配置 AI 时才使用本地规划器降级生成。

单个正方形、圆形、三角形、六边形、五角星，以及节点增删改、连线等基础绘制由 Simple Path 本地即时执行，不调用 LLM。

`画/绘制/生成`单个或多个基础图形会新建空白画布；`添加/加/放置`会把图形追加到当前画布。新图或完整图表替换当前画布前，系统会自动备份上一张图；也可以随时点击画布顶部“保存当前图”。

### 专业图表与自由画图

- 专业图表模式支持流程图、用例图、组织结构图、架构图、思维导图、数据流图、框架图和表格，并支持添加图形、修改文字与颜色、连接节点和调整相对位置。
- 自由画图模式使用独立 SVG Scene。花朵、杯子等常见物体走本地快速绘制，未预设物体由 LLM 拆分为经过安全校验的可编辑 SVG 基础图元。
- 删除、清空和模式切换等确定性命令本地直通；当 Agent 完全无法确定修改目标或存在关键语义歧义时，会通过可见反问窗口等待用户继续回答。

### 标准矢量导出

PNG 导出适合展示与分享；专业图表和自由画布的 SVG 导出均直接序列化为标准矢量 XML，不依赖 `foreignObject` 网页截图。导出文件包含真实节点、图形、文字、连线和箭头，可以无损缩放并继续编辑。

## 题目要求映射

| 题目要求           | VoiceFlow 实现                                    |
| ------------------ | ------------------------------------------------- |
| 纯语音绘图         | 只读 React Flow 画布，绘图操作统一由语音命令驱动  |
| 指令理解准确与容错 | 本地 ASR 校准、当前画布上下文匹配、确定性最佳匹配 |
| 降低语音到绘图延迟 | 四级路由、Fast Path 本地直通、实时延迟指标        |
| 复杂指令拆解与执行 | Agent 输出经过校验的 Operation 批次并直接提交     |
| 可恢复与安全       | 撤销/重做、持久版本、运行时 Diagram 与 SVG 校验   |

## 真实 AI 配置

完整结构图生成和无法由本地规则确定的上下文修改需要真实 OpenAI-compatible Provider；基础图形、本地校准与确定性编辑无需配置 AI：

```bash
copy .env.example .env.local
npm install
npm run dev
```

请在 `.env.local` 中填写 `VITE_AI_BASE_URL`、`VITE_AI_API_KEY` 和 `VITE_AI_MODEL`。未配置时，Fast、Simple、Workflow 和本地语音校准仍可使用；Agent 请求会明确提示真实 AI 尚未配置。浏览器中的 `VITE_` 配置仅适合本地演示，生产环境应使用后端代理。

## 推荐语音命令

- Fast Path：`撤销`、`重做`、`保存`、`导出 SVG`、`看全图`
- 本地语音校准：`声成一张强化学西的流成图`
- 确定性修改：`把失败分支改成红色虚线`
- 通用结构图：`画一个学生选课用例图`、`画一个公司的组织结构图`、`做一个方案对比表格`
- 自由画图：`切换到自由画图模式`、`画一棵绿色的小树`、`删除小树`、`清空自由画布`
- Workflow：`整理成适合汇报的版本`、`保存当前版本叫修改完成`

## 工程质量

- Diagram JSON 是唯一绘图数据源，组件不直接修改图表。
- AI 输出、Operation 和持久化版本进入状态前均经过运行时校验。
- React Flow 画布延迟加载，保持首屏主包体积可控。
- GitHub Actions 自动执行完整质量检查。

```bash
npm run typecheck
npm run lint
npm run format:check
npm test
npm run build
```
