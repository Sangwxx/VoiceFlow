# VoiceFlow：纯语音控制的智能绘图工具方案设计书

## 1\. 项目背景

在流程图、架构图、业务图、系统图等绘图场景中，用户通常需要通过鼠标拖拽节点、手动连接线条、反复调整布局、修改样式和导出文件。对于非专业设计人员来说，绘图工具的操作成本较高；对于会议讨论、头脑风暴、教学演示、方案汇报等场景来说，手动绘图效率也较低。

本项目拟开发一款纯语音控制的智能绘图工具 VoiceFlow。用户无需使用鼠标或键盘，仅通过自然语音即可完成图形创建、节点修改、连线调整、自动排版、主题美化、版本回退和文件导出等操作。

本项目重点是“AI 生成可编辑图结构”。系统会将用户语音转化为结构化 Diagram JSON，再由前端画布实时渲染为可编辑流程图或架构图。最终图形不是静态图片，而是由节点、连线、分组、样式和布局组成的可维护图结构。

## 2\. 项目定位

### 2\.1 产品名称

VoiceFlow：纯语音驱动的智能流程图与架构图绘制工具

### 2\.2 产品一句话介绍

VoiceFlow 是一款通过语音创建和编辑可视化图形的 AI 绘图工具，支持用户仅通过语音完成流程图、架构图等结构化图形的绘制、修改、排版、美化、撤销和导出。

### 2\.3 核心定位

本产品不是传统绘图软件，也不是 AI 生图工具，而是：
一个语音驱动的可编辑图形生成与操作系统。

系统将用户语音拆分为两类：

1. 高频确定命令，例如撤销、重做、保存、导出、放大、暂停等，直接由快捷命令引擎执行。

2. 复杂绘图命令，例如“画一个电商下单流程图”“把这个流程整理成汇报版”等，由 AI 规划器解析为结构化图形操作。

## 3\. 设计目标

### 3\.1 功能目标

VoiceFlow 需要支持以下能力：

1. 用户通过语音新建流程图或系统架构图。

2. 系统自动解析用户语音中的节点、连线、分支、层级和布局方向。

3. 系统自动生成 Diagram JSON，并渲染为可编辑图形。

4. 用户可以通过语音继续添加、删除、修改节点和连线。

5. 用户可以通过语音调整图形布局，例如从上到下、从左到右、自动排版。

6. 用户可以通过语音修改样式，例如改颜色、改虚线、加粗主流程。

7. 用户可以通过语音执行撤销、重做、保存、导出等高频操作。

8. 系统具备歧义识别和澄清能力。

9. 系统具备版本管理能力，支持回到上一步、回到某个版本、保存当前版本。

10. 系统支持导出 SVG、PNG 和 JSON，后续可扩展 draw\.io XML、PPT 等格式。

### 3\.2 交互目标

1. 用户全程不使用鼠标和键盘。

2. 高频命令响应速度要快，撤销、暂停、保存、放大等命令应接近快捷键体验。

3. 复杂命令需要展示系统理解结果，使用户知道系统准备执行什么。

4. 系统在低置信度情况下不盲目执行，而是主动追问。

5. 用户可以随时打断系统当前操作。

6. 用户可以边说边改，避免一次性生成后不满意重来的低效体验。

### 3\.3 工程目标

1. 架构清晰，便于多人协作。

2. 模块独立，便于拆分 PR。

3. 数据结构稳定，便于测试和调试。

4. 绘图结果可复现、可回退、可导出。

5. AI 只负责理解和规划，不直接操作底层画布。

6. 前端渲染、布局、状态管理、版本管理都可控。

## 6\. 核心创新设计

### 6\.1 双通道语音控制架构

传统方案流程：语音 → 文本 → AI 分析 → 绘图，存在延迟高、指令易被过度解析、成本高、体验不稳定等问题。

VoiceFlow 采用双通道架构：

- **Fast Path**：高频确定命令直接执行，无 AI 参与

- **Agent Path**：复杂绘图需求进入 AI 规划流程

**示例**
用户说：撤销 → 系统直接执行 `history.undo()`
用户说：帮我画一个电商下单流程图，并补全异常分支 → 进入 AI 规划，生成 Diagram JSON

该设计解决响应延迟、指令准确性、操作稳定性、复杂任务拆解四大问题。

### 6\.2 Diagram JSON 中间层

系统不让 AI 直接操作画布，而是先生成统一图结构。

**示例结构**

```JSON
{
  "diagram_type": "flowchart",
  "title": "用户登录流程",
  "nodes": [
    {
      "id": "start",
      "label": "开始",
      "type": "start"
    },
    {
      "id": "open_app",
      "label": "打开 App",
      "type": "process"
    },
    {
      "id": "check_login",
      "label": "是否已登录？",
      "type": "decision"
    },
    {
      "id": "home",
      "label": "首页",
      "type": "process"
    }
  ],
  "edges": [
    {
      "id": "e1",
      "from": "start",
      "to": "open_app"
    },
    {
      "id": "e2",
      "from": "open_app",
      "to": "check_login"
    },
    {
      "id": "e3",
      "from": "check_login",
      "to": "home",
      "label": "是"
    }
  ],
  "layout": {
    "direction": "top_down"
  },
  "theme": {
    "name": "business_blue"
  }
}
```

**优势**：可编辑、可导出、可撤销、可版本管理、可测试、可解释、可迁移至其他绘图工具。

### 6\.3 Operation JSON 操作层

后续修改不重新生成整张图，仅生成局部操作指令。

**示例1**：把登录页改成蓝色

```JSON
{
  "action": "update_node_style",
  "target": {
    "node_id": "login_page"
  },
  "patch": {
    "background": "#DCEBFF",
    "border": "#2F80ED"
  }
}
```

**示例2**：在登录页后面加一个验证码校验

```JSON
{
  "action": "insert_node_after",
  "target": {
    "node_id": "login_page"
  },
  "new_node": {
    "id": "captcha_check",
    "label": "验证码校验",
    "type": "process"
  }
}
```

该设计规避了“每次修改都重画整张图”的弊端。

### 6\.4 语音理解回显与置信度容错

系统对语音解析结果做置信度分级处理：

- 高置信度：直接执行

- 中置信度：执行前回显操作内容

- 低置信度：主动追问用户

**示例**

1. 高置信度（撤销）：系统回复「已撤销」

2. 中置信度（把失败分支改成红色）：系统回复「已将“登录失败”分支改成红色」

3. 低置信度（把失败分支删掉，存在多条失败分支）：系统回复「当前有“登录失败”和“支付失败”两条失败分支，你要删除哪一条？」

### 6\.5 汇报美化模式

用户说出「把这个流程改成适合汇报的版本」，系统自动执行：

1. 精简节点文案

2. 主流程高亮

3. 异常分支弱化

4. 统一节点宽度

5. 统一字体和颜色

6. 调整节点间距

7. 增加标题

8. 自动居中

9. 应用商务主题

### 6\.6 版本时间旅行

支持语音操控版本：撤销、重做、回到指定版本、命名保存版本、版本对比等。系统记录每一步操作并生成快照。

**版本记录示例**

```JSON
{
  "version_id": "v5",
  "name": "自动美化前",
  "timestamp": "2026-06-12 20:30:00",
  "operation": "apply_business_theme",
  "snapshot": {
    "nodes": [],
    "edges": [],
    "layout": {},
    "theme": {}
  }
}
```

## 7\. 功能模块详细设计

### 7\.1 语音输入模块 Voice Input

#### 7\.1\.1 功能说明

负责采集用户语音，并将语音转化为文本。

#### 7\.1\.2 功能要求

1. 支持开始监听、停止监听

2. 支持实时语音字幕

3. 支持语音输入结束判断、连续语音输入

4. 支持用户打断当前任务

5. 支持中文普通话，后续扩展中英混合术语

#### 7\.1\.3 示例

用户说：画一个用户登录流程图
系统识别文本：画一个用户登录流程图

#### 7\.1\.4 UI 展示

界面底部显示语音状态：
`正在聆听……`
`识别结果：画一个用户登录流程图`

#### 7\.1\.5 技术实现

第一版：Web Speech API
后续增强：Whisper Streaming

#### 7\.1\.6 容错设计

1. 语音识别失败，提示用户重说

2. 仅识别到部分语句，等待用户补充

3. 实时监听“暂停”“停止”“取消”等高优先级指令

### 7\.2 指令路由模块 Command Router

#### 7\.2\.1 功能说明

判断用户语音所属命令类型，分发至不同处理路径。

#### 7\.2\.2 三类路径

- Fast Path：快捷命令

- Simple Path：简单绘图命令

- Agent Path：复杂绘图命令

#### 7\.2\.3 判断规则

- Fast Path：撤销、重做、保存、导出、暂停、放大、缩小、看全图等

- Simple Path：加节点、删除节点、修改颜色、添加连线等

- Agent Path：绘制完整流程图/架构图、补全分支、转为汇报版、转为泳道图等

#### 7\.2\.4 输出格式

```JSON
{
  "route": "fast_path",
  "command": "undo",
  "confidence": 0.98
}
```

```JSON
{
  "route": "agent_path",
  "intent": "create_complex_diagram",
  "confidence": 0.91
}
```

#### 7\.2\.5 创新点

该模块是产品低延迟、高稳定性的核心，规避全部命令调用大模型，提升响应速度与准确率。

### 7\.3 快捷命令模块 Fast Command Engine

#### 7\.3\.1 功能说明

处理高频、确定、低风险的语音命令，**不调用大模型**。

#### 7\.3\.2 支持命令

- 历史操作类：撤销、重做、回到上一步、恢复刚才、回到上一版

- 视图控制类：放大、缩小、看全图、居中、适应屏幕

- 系统控制类：暂停、继续、停止、取消、确认

- 保存导出类：保存、保存当前版本、导出 SVG、导出 PNG、导出 JSON

#### 7\.3\.3 响应目标

- 暂停/取消：50\-100ms

- 撤销/重做：100\-200ms

- 放大/缩小：50\-100ms

- 保存/导出：200\-500ms

#### 7\.3\.4 示例

用户说：撤销 → 系统执行 `history.undo()`
用户说：看全图 → 系统执行 `viewport.fitView()`

### 7\.4 简单绘图命令模块 Simple Command Parser

#### 7\.4\.1 功能说明

处理简单的图形增删改命令，生成 Operation JSON。

#### 7\.4\.2 支持能力

1. 添加/删除节点

2. 修改节点文字、颜色

3. 修改连线样式、添加连线

4. 插入节点、调整节点位置

5. 重新排版

#### 7\.4\.3 示例

用户说：加一个节点，叫验证码校验

```JSON
{
  "action": "create_node",
  "node": {
    "id": "captcha_check",
    "label": "验证码校验",
    "type": "process"
  }
}
```

```JSON

  },

    "label": "验证码校验",
    "type": "process"
```

#### 7\.4\.4 容错设计

1. 找不到目标节点：主动询问用户是否指向近似节点

2. 匹配到多个节点：列出节点列表，让用户选择

### 8\.8 语音打断

#### 场景

#### 系统处理

1. 高优先级监听器捕获指令

3. 反馈：已停止当前操作

### 8\.9 歧义澄清

当前有三个失败分支：登录失败、支付失败、库存不足。你要删除哪一个？
用户选定后，系统执行对应删除操作。

## 11\. 指令分类设计

### 11\.1 Fast Path 指令

撤销、重做、暂停、继续、保存、导出、放大、缩小、看全图、居中、确认、取消

### 11\.2 Simple Path 指令

加一个节点、删除某个节点、修改某个节点名称、把某个节点改成蓝色、连接 A 到 B、把某条线改成虚线

### 11\.3 Agent Path 指令

画一个完整流程图、画一个系统架构图、自动补全异常分支、把图改成汇报版、把图改成技术文档版、检查流程有没有缺失、把流程改成泳道图

## 12\. 响应延迟设计

### 12\.1 分级响应目标

- P0 紧急命令：50\-100ms

- P1 快捷命令：100\-200ms

- P2 简单编辑：300\-800ms

- P3 新增节点：500ms\-1s

- P4 复杂生成：2\-8s

- P5 自动美化：2\-5s

### 12\.2 延迟优化策略

1. 高频命令不走 AI

2. 简单命令优先规则解析

3. 复杂命令才调用 LLM

4. 先生成图结构，再逐步渲染

5. 长任务支持暂停、取消

6. 任务执行中可接收新指令

7. 操作队列分批执行，防止 UI 卡死

## 13\. 容错设计

### 13\.3 多对象歧义

匹配到多个目标时，列出选项供用户选择。

### 13\.4 误操作恢复

支持语音调用撤销、回到上一步等指令恢复。

### 13\.5 AI 输出不合法

## 19\. 风险与应对

### 19\.1 语音识别不稳定

应对：实时字幕、支持重识别、建立指令同义词库、后期升级 ASR 能力。

### 19\.2 AI 输出 JSON 不稳定

应对：JSON Schema 强校验、自动修复简单错误、异常时保留原图状态。

### 19\.3 复杂布局效果不佳

应对：初期限定流程图/架构图、基础布局使用 Dagre、复杂场景升级 ELK\.js、支持语音重新排版。

### 19\.4 功能范围过大

应对：首期仅实现流程图\+架构图、聚焦核心闭环、不做自由手绘、不替代专业 CAD/设计软件。

VoiceFlow 技术组合：
React / Next\.js \+ React Flow \+ Dagre \+ Zustand \+ Web Speech API / Whisper \+ LLM \+ Diagram JSON \+ Fast Command Layer \+ Agent Planner \+ 自动美化与版本管理

本项目核心并非 AI 直接绘图，而是将语音转化为标准化结构化图形语言，再由前端可编辑画布完成渲染。双通道架构兼顾高频操作的低延迟与复杂任务的智能化。

最终产品实现：用户仅依靠语音，即可完成流程图/架构图的创建、编辑、排版、美化、撤销、导出全流程，打造出一套完整、流畅、可解释、可扩展的纯语音驱动可编辑图形操作系统。

**一句话总结**：
VoiceFlow 是一个纯语音驱动的可编辑图形操作系统，而不是简单的语音转文字绘图工具。
