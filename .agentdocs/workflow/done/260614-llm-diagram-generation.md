# LLM 主导通用图表生成

## 背景

当前完整图表主要由本地结构预设生成，稳定但主题适应性不足。基础图形命令也需要明确与完整图生成分流，保证简单绘制即时、可靠。

## 目标

- 完整图表由 LLM 根据受约束的结构协议生成，本地负责验证、样式补全与自动布局。
- 单个图形、文字、节点和连线继续使用 Simple Path 本地执行，不调用 LLM。
- 保留本地规划器作为未配置 AI 或 AI 请求失败时的明确降级能力。
- 文字测试入口复用语音最终指令链路。
