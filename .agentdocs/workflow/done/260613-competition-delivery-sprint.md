# 参赛交付冲刺

## 截止时间

2026 年 6 月 14 日晚，需要同时提交项目与 Demo 视频。

## 交付目标

围绕“AI 语音绘图工具”题目，在不进行高风险架构重写的前提下，让评委能够直接看到并理解项目对语言歧义和响应延迟的解决方案。

## 范围约束

- 保留现有 Fast、Simple、Workflow、Agent 四级路由架构。
- 保留现有候选澄清、最近目标消歧、语义纠错和 Proposal 确认机制。
- 不引入新的后端、云端语音识别或外部服务。
- Mock AI 模式必须能够完成完整核心演示，避免录制期间受网络波动影响。
- 优先实现评委可见、可量化、可稳定录制的功能。

## 分阶段计划

- [x] 阶段一：确定最小交付范围与录制风险。
- [x] 阶段二：实现延迟路径仪表盘与可解释消歧展示。
- [x] 阶段三：完善参赛版 README、GitHub Actions 和录制脚本。
- [x] 阶段四：执行完整验证并完成视频录制清单。

## 核心叙事

VoiceFlow 通过“确定性命令本地快速执行、复杂意图交给 Agent、低置信度主动澄清、危险修改先预览确认”的分层策略，同时解决纯语音绘图中的响应延迟与语言歧义问题。

## 最终结果

- 完成 Fast Path 与四类路由的真实延迟指标展示。
- 完成高、中、低三级置信度语义纠错。
- 完成纯语音可选的视觉化消歧候选框。
- 完成中文参赛 README、三分钟 Demo 脚本、设计审计、界面截图和 GitHub Actions。
- 四项交付分别通过功能分支提交并推送，随后合并到 `main`。
